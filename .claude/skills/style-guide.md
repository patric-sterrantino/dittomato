# vialytics Content Style Guide

## Inclusive Language
- Use person-first language: "users with disabilities" not "disabled users"
- Use gender-neutral language: singular "they", avoid "he/she" or "Dear Sir/Madam"
- Avoid idioms and colloquialisms — use literal language that translates well across cultures
  - DO: "It's straightforward", "Please wait"
  - DON'T: "It's a piece of cake", "Hang tight"

## Word List (English source terms)
Use the preferred form — apply the same principle when choosing the equivalent in other languages:

| Preferred       | Avoid              | Note                                                  |
|-----------------|--------------------|-------------------------------------------------------|
| after           | once               | "once" is ambiguous with "one time"                   |
| because         | since              | "since" is ambiguous with time                        |
| email           | e-mail             | One word, no hyphen, lowercase                        |
| homepage        | home page          | One word                                              |
| log in (v) / login (n/adj) | Login to your account | Verb = two words, noun/adjective = one word |
| never mind      | nevermind          | Two words                                             |
| OK              | okay / ok          | Always "OK"                                           |
| opt in (v) / opt-in (n/adj) | Opt-in to notifications | Verb = two words, noun = hyphenated |
| sign up (v) / signup (n/adj) | Signup for free | Verb = two words, noun/adjective = one word |
| URL             | url                | All caps                                              |
| username        | user name          | One word, lowercase                                   |
| website         | web site           | One word, lowercase                                   |

## General Tone
- Concise and unambiguous — no filler, no padding
- Sentence case unless the source uses Title Case
- Active voice, direct phrasing
- Use literal language that works across cultures
